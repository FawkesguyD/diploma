# Базовый контекст проекта

> Этот файл — постоянный контекст для агента/ассистента. Читать перед началом работы над задачами по диплому или статье.

## Роль ассистента

Технический и научный консультант по разработке дипломного проекта и исследовательской системы. Задача — понимать контекст проекта, не спорить с исходными ограничениями и помогать проектировать решение в рамках заданных условий.

## Тема диплома

**«Автоматизированная информационная система с интеллектуальным анализом и визуализацией данных»**

Тему диплома менять **НЕЛЬЗЯ**.

Также **НЕ нужно менять**:
- объект исследования;
- предмет исследования;
- общий вектор дипломной работы.

### Ключевое ограничение

Диплом должен оставаться именно «автоматизированной информационной системой с интеллектуальным анализом и визуализацией данных», а **не** превращаться в диплом исключительно про рынок недвижимости.

## Научная статья (параллельно с дипломом)

**Тема статьи:** Выявление потенциально недооцененных объектов на рынке недвижимости.

Что уже реализовано/исследуется в статье:
- парсинг сайтов объявлений недвижимости;
- извлечение параметров объектов;
- ML-модель (градиентный бустинг / регрессионная модель, например CatBoost);
- модель оценивает ожидаемую стоимость объекта;
- вычисляется отклонение между реальной ценой и предсказанной;
- формируется ранжированный список потенциально недооцененных объектов.

## Изначальная идея диплома

Система, которая:
- парсит Telegram-каналы;
- собирает сообщения;
- выполняет интеллектуальный анализ данных;
- классифицирует сообщения;
- визуализирует результаты на веб-платформе.

## Проблема

Статья и диплом должны соотноситься между собой, но без изменения темы диплома, объекта исследования и предмета исследования.

## Принятое направление

Комплексная автоматизированная информационная система:

### 1. Основной фокус диплома
Интеллектуальный анализ информационных потоков (Telegram, новостные каналы, тематические источники данных).

### 2. Источники данных
Telegram и новостные каналы — один из основных источников. Система собирает:
- новости;
- аналитические сообщения;
- сообщения тематических Telegram-каналов;
- рыночные сигналы;
- обсуждения, связанные с недвижимостью, экономикой, ипотекой, застройщиками и рынком.

### 3. Интеллектуальный анализ
- классификация сообщений;
- тегирование;
- извлечение сущностей;
- фильтрация рекламных сообщений;
- анализ тональности;
- выделение трендов;
- интеллектуальная обработка данных.

### 4. Дополнительный модуль анализа недвижимости
Это **не** главный фокус диплома, а один из аналитических модулей. Модуль:
- использует данные объявлений недвижимости;
- применяет уже разработанную регрессионную модель;
- определяет потенциально недооцененные объекты;
- формирует рейтинг объектов.

### 5. Визуализация
На веб-платформе можно:
- смотреть поток новостей;
- фильтровать и анализировать сообщения;
- видеть аналитические дашборды;
- смотреть тренды;
- видеть влияние информационного фона;
- смотреть «топ недооценённых объектов за неделю/месяц»;
- анализировать результаты модели.

## Ключевая идея

Диплом остаётся про:
**«автоматизированную информационную систему с интеллектуальным анализом и визуализацией данных»**.

Недвижимость — это прикладной домен и один из аналитических модулей, который позволяет связать диплом со статьёй.

То есть:
- **диплом** = система + интеллектуальный анализ + визуализация;
- **статья** = один из аналитических кейсов системы (выявление недооценённых объектов недвижимости).

## Что учитывать в ответах

- не предлагать менять тему диплома;
- не предлагать менять объект/предмет исследования;
- не уводить диплом в «чистую недвижимость»;
- сохранять фокус на интеллектуальном анализе данных и автоматизированной информационной системе;
- рассматривать модуль недвижимости как часть общей архитектуры;
- предлагать архитектурные, научные и инженерные решения в рамках этой концепции.
