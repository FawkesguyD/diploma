# Документация проекта

> Дипломный проект: **«Автоматизированная информационная система с интеллектуальным анализом и визуализацией данных»**.

Эта папка — единая точка входа в проектную документацию. Начни отсюда.

## Карта документации

| Раздел | Что внутри | Когда читать |
|---|---|---|
| [architecture/](architecture/) | PlantUML-диаграммы (context / container / component / deployment) | Чтобы понять, **как устроена** система |
| [services/](services/) | Человекочитаемые описания каждого сервиса | Чтобы понять, **что делает** конкретный сервис |
| [decisions/](decisions/) | ADR — почему сделано именно так | Чтобы понять, **почему** мы выбрали этот вариант |
| [design/](design/) | Проектные артефакты (поток 1 плана): схемы БД, контракты очередей, OpenAPI | Чтобы понять **детали реализации** |

## Принципы поддержки документации

1. **Диаграмма архитектуры — источник правды.** Любое изменение в системе сначала отражается в [architecture/](architecture/), потом в коде.
2. **Каждый сервис — README по шаблону** (см. [services/README.md](services/README.md)). Целевая аудитория: научный руководитель и комиссия.
3. **Каждое нетривиальное архитектурное решение — ADR.** Это страховка от потери контекста.
4. **Документация — не план «как хотелось бы», а описание текущего состояния.**

## Быстрая навигация по архитектуре

- Хочу увидеть **общую картину** → [architecture/container.puml](architecture/container.puml)
- Хочу понять **развёртывание** → [architecture/deployment.puml](architecture/deployment.puml)
- Хочу понять **внутренности сервиса** → [architecture/component.puml](architecture/component.puml) + соответствующий файл в [services/](services/)
- Хочу понять **почему так** → [decisions/](decisions/)
