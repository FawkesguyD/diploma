Этот раздел — то, что фиксируется **перед** разработкой и связывает все потоки работы. Контракты здесь — источник правды; код в `services/*` обязан им соответствовать.

| Раздел | Что внутри |
|---|---|
| [scope.md](scope.md)                  | Объект, предмет, цели АИС, задачи диплома, что НЕ входит, словарь терминов. |
| [databases/](databases/)              | Схемы PostgreSQL / MongoDB / ClickHouse. |
| [messaging/](messaging/)              | Контракты RabbitMQ (команды) и Kafka (метрики). |
| [api/](api/)                          | OpenAPI: REST-контракт между фронтом и шлюзом. |
| [dashboards.md](dashboards.md)        | Функциональная спецификация дашбордов. |
| [acceptance.md](acceptance.md)        | Чек-лист готовности к защите (MVP / Target / Stretch). |

## Порядок чтения для нового участника

1. [CONTEXT.md](../../CONTEXT.md) в корне — что за проект, какие ограничения.
2. [scope.md](scope.md) — формальная шапка.
3. [docs/architecture/](../architecture/) — диаграммы C4 (контекст → контейнеры → компоненты → деплой).
4. [docs/decisions/](../decisions/) — ADR. Объясняют **почему** именно так.
5. Этот раздел — **как именно** устроены данные и контракты.
