# ADR-0001: Три БД — PostgreSQL + MongoDB + ClickHouse

- **Дата:** 2026-05-14
- **Статус:** accepted

## Контекст

Система обрабатывает данные трёх принципиально разных профилей:

1. **Реляционные справочные данные**: пользователи, источники, настройки модулей, метаданные запусков моделей. Нужны транзакции и FK.
2. **Документные данные**: сообщения Telegram и новостных каналов, объявления недвижимости. Гетерогенная структура, много необязательных полей, JSON-нативно.
3. **Аналитика и метрики**: агрегаты по темам/времени/регионам для дашбордов. Тяжёлые `GROUP BY` по миллионам строк, запись в режиме append-only.

Если использовать одну БД для всего, либо проигрываем в производительности (Postgres на агрегациях по миллионам метрик), либо в эргономике (Mongo для пользователей и FK — неудобно), либо в выразительности (ClickHouse как первичное хранилище — слабые update/transactions).

## Решение

Три специализированных БД, каждая под свой профиль нагрузки:

- **PostgreSQL** — пользователи, конфиги, справочники, `model_runs`, `parser_jobs`.
- **MongoDB** — `messages`, `realestate_listings`, `nlp_annotations`.
- **ClickHouse** — таблицы метрик и агрегатов (`messages_by_topic_hourly`, `prices_timeseries`, `district_activity`).

## Альтернативы

- **Только PostgreSQL** (с jsonb и партиционированием для метрик) — проще в эксплуатации, но проигрывает ClickHouse на агрегациях ×10–×100 и хуже работает с гетерогенными документами.
- **Только MongoDB** — неудобно для users/конфигов с FK, нет аналитической производительности.
- **Postgres + ClickHouse без Mongo** — пришлось бы хранить сырые сообщения как jsonb в Postgres, потеряли бы простоту document-first модели для парсера и NLP.

## Последствия

**+** Каждая БД работает в своей сильной зоне.
**+** Изоляция нагрузок: тяжёлая аналитика не мешает оперативной работе.
**+** Подсистема метрик/дашбордов независима от прикладных сервисов.

**−** Оперировать тремя БД сложнее: бэкап, миграции, мониторинг — отдельно для каждой.
**−** Дублирование инфраструктуры в `docker-compose`.
**−** Нужны init-скрипты для каждой (alembic для PG, JS init для Mongo, DDL для CH).
