# Документация сервисов

Здесь — человекочитаемая документация по каждому сервису системы. Целевая аудитория: научный руководитель, члены комиссии и ты сам через полгода.

## Сервисы системы

| Сервис | Документ | Краткое описание |
|---|---|---|
| Realestate | [realestate.md](realestate.md) | Оценка стоимости и ранжирование недооценённых объектов недвижимости (CatBoost) |
| NLP / Parser | [nlp-parser.md](nlp-parser.md) | Парсинг Telegram и новостей + интеллектуальный анализ сообщений |

> Документы по сервисам создаются по мере их проектирования/разработки. Шаблон ниже.

## Шаблон документа сервиса

Сохрани как `services/<имя>/README.md` или `docs/services/<имя>.md`:

```markdown
# <Название сервиса>

## Назначение

1–2 абзаца человеческим языком: зачем этот сервис существует, какую
проблему решает в общей системе. Без жаргона.

## Ответственности

**Делает:**
- ...
- ...

**НЕ делает** (важно для границ модуля):
- ...
- ...

## Внешние интерфейсы

### HTTP API

| Метод | Путь | Описание |
|---|---|---|
| GET | `/health` | Проверка живости |
| POST | `/predict` | ... |

### Очереди / топики

**Читает:**
- `queue.name` (RabbitMQ) — формат сообщения, описание

**Пишет:**
- `topic.name` (Kafka) — формат, частота

## Зависимости

- **PostgreSQL** — для чего используется
- **MongoDB** — какие коллекции
- **Другие сервисы** — какие вызовы

## Данные

| Сущность | Хранилище | Назначение |
|---|---|---|
| ... | PostgreSQL.users | ... |

## Конфигурация

Переменные окружения (см. `.env.example`):

| Переменная | Пример | Описание |
|---|---|---|
| `DATABASE_URL` | `postgres://...` | Подключение к Postgres |

## Запуск локально

```bash
# через docker compose из корня репозитория:
docker compose up -d <service-name>

# напрямую:
cd services/<name>
poetry install
poetry run uvicorn app.main:app --reload --port 8001
```

## Как протестировать

Smoke-сценарий:

```bash
curl http://localhost:8001/health
# ожидаем: {"status": "ok"}

curl -X POST http://localhost:8001/predict \
  -H "Content-Type: application/json" \
  -d '{"area": 50, "rooms": 2, ...}'
# ожидаем: {"predicted_price": 12500000, "confidence": 0.87}
```

## Как пользоваться шаблоном

1. Скопируй шаблон в новый файл.
2. Заполняй разделы по мере появления функциональности — пустые секции лучше пометить `_(TBD)_`, чем удалять.
3. После любого изменения API/контракта — синхронно обнови документ и схему в [../architecture/](../architecture/).
4. Документ должен читаться как пояснительная записка — без отсылок «см. в коде».
